Introduction

Background

  • Air pollution is a major environmental and public health challenge globally
  • Riau province in Indonesia has the largest peatland in Sumatra (4,044 million hectares)
  • Illegal forest burning causes significant air pollution and thick smog annually
  • Current monitoring systems (ISPU) are limited in:
    • Fixed installation points with limited coverage
    • Non-real-time updates (only at 15:00 WIB each day)
    • Slow data processing and management

Research Objectives

  • Develop a real-time air quality monitoring system based on IoT
  • Create an accessible web interface to display air quality information
  • Measure multiple air quality parameters: temperature, humidity, CO, CO2, and alcohol presence
  • Enable remote access to air quality data for public awareness

"IoT technology enables transferring data over a network without requiring human-to-human or human-to-computer interaction, creating systems that can sense and connect to their environment."

Internet of Things (IoT)

A network of physical devices connected to the internet that can collect and exchange data without human intervention, providing real-time insights and enabling smarter decision-making.

Related Works

Previous Research Overview

The researchers reviewed five previous studies to establish the foundation for their work, identifying gaps and opportunities in IoT-based environmental monitoring systems.

  • Most previous studies focused on either air, water, or soil quality individually
  • Limited research on systems that combine multiple environmental parameters
  • Lack of real-time web interfaces for public information dissemination
  • Need for improved accessibility and user-friendly visualization

Fuzzy Logic Approach

Prayudha et al. (2018) combined IoT with fuzzy intelligence methods for air quality measurement, achieving more accurate results than conventional methods but requiring high-speed internet connectivity.

Water Quality Monitoring

Yusuf (2018) implemented IoT technology for water quality monitoring, measuring temperature, turbidity, and oxygen levels in the Cimahi river using Arduino Uno.

Mobile Application Integration

Waworundeng & Lengkong (2018) used the Wemos microcontroller with the MQ135 sensor to notify users about indoor air quality through the Blynk mobile application.

Automated Plant Irrigation

Putri et al. (2019) designed an IoT-based automatic plant sprinkler for miniature greenhouses using the DHT11 sensor to monitor humidity and temperature to automate watering systems.

Research Gap Identified

Previous research had not studied an air quality monitoring system that combined measuring CO, CO2, alcohol, temperature, and air humidity using an IoT-based website as an interface. This opportunity led to the development of the proposed system.

Methodology

Sensors & Hardware

Component Description Function in System
DHT11 Sensor Temperature and humidity sensor Measures ambient temperature (°C) and humidity (%)
MQ135 Sensor Air quality gas sensor Detects CO, CO2, alcohol, ammonia, NOx, benzene (in ppm)
Arduino Uno Microcontroller board Interfaces with sensors, processes analog outputs from MQ135
Raspberry Pi Single-board computer Receives data from Arduino, connects to the internet, sends data to server

Software Development Approach

  • Waterfall Method: Structured and sequential process used for web development
  • Requirements Definition: Identifying system needs and functionality
  • System and Software Design: Creating the architecture for hardware and web interface
  • Implementation: Coding and hardware assembly
  • Integration & Testing: Connecting components and verifying functionality
  • Operation & Maintenance: Deployment and ongoing management

Air Quality Assessment Categories

  • Good: 0-50 ppm average contaminant level
  • Moderate: 51-100 ppm average contaminant level
  • Unhealthy: 101-199 ppm average contaminant level
  • Very Unhealthy: 200-299 ppm average contaminant level
  • Dangerous: 300-500 ppm average contaminant level
  • Temperature Assessment:
    • 24-27°C: Normal (green indicator)
    • Above 27°C: Hot/Abnormal (red indicator)

Measurement Scenario

The research team designed a testing methodology by placing the sensor in "Good" condition areas, then gradually introducing smoke containing CO and alcohol to observe changes through different air quality levels. Tests were conducted at different times to validate temperature and humidity measurements, which naturally vary throughout the day.

System Design

System Architecture

  • DHT11 & MQ135 sensors connect to Arduino Uno
  • Arduino Uno connects to Raspberry Pi via serial connection
  • Raspberry Pi processes data and connects to the internet
  • Data is sent to MySQL database on a web server
  • Web interface displays results in real-time

Implementation Details

  • Arduino Uno handles analog output from MQ135 sensor
  • Raspberry Pi receives data through serial connection
  • Wi-Fi connection used for wireless data transmission
  • Real-time data synchronization with MySQL database
  • Web interface developed for visualization and user access
  • Air quality categorization based on averaged sensor data

System Flowchart

The flowchart illustrates the sequential process of data collection, processing, and display:

  1. DHT11 sensor detects temperature and humidity
  2. MQ135 sensor detects air contaminants (CO, CO2, alcohol)
  3. Data is sent to Arduino Uno for initial processing
  4. Processed data is transferred to Raspberry Pi
  5. Raspberry Pi sends data to web server via internet
  6. Web interface displays monitoring results for users

Hardware Configuration

The hardware assembly connects sensors to the processing units:

  • DHT11 connected to Arduino digital pins
  • MQ135 connected to Arduino analog pins
  • Arduino Uno connected to Raspberry Pi via USB
  • Raspberry Pi configured with internet connectivity
  • Power supplied to ensure continuous operation

Conclusion & Future Work

Key Findings

  • Successfully developed a real-time air quality monitoring system using IoT technology
  • Combined DHT11 and MQ135 sensors to measure both environmental conditions and air contaminants
  • Integrated Arduino Uno and Raspberry Pi to process and transmit data to a web server
  • Created a web interface displaying temperature, humidity, CO, CO2, and alcohol levels
  • Implemented categorization system with color-coded indicators for different air quality levels
  • Demonstrated accurate detection of changing air quality conditions in testing scenarios

Advantages Over Existing Systems

Real-Time Monitoring

Unlike government ISPU systems that update only once daily at 15:00 WIB, this system provides continuous real-time data updates.

Remote Accessibility

Users can access air quality information from anywhere via the web, not just at fixed monitoring stations.

Multi-Parameter Analysis

Combines temperature and humidity with air contaminant measurements for more comprehensive air quality assessment.

Recommendations for Future Research

Sensor Enhancements

  • Incorporate additional sensors for PM10, PM2.5, O3, NO2, and other air pollutants
  • Use more sensitive sensors compatible with Arduino for improved accuracy
  • Develop better calibration methods for sensor readings
  • Implement redundant sensor arrays for reliability
  • Explore energy-efficient sensor technologies for battery operation

System Improvements

  • Develop more accurate air quality calculation algorithms
  • Create mobile applications for enhanced user accessibility
  • Implement predictive analytics for air quality forecasting
  • Add notification systems for alerting users about poor air quality
  • Establish networks of devices for citywide air quality mapping

Final Assessment

The research successfully demonstrated a practical implementation of IoT technology for air quality monitoring that addresses limitations in existing systems. With further development and sensor additions, this approach could provide valuable public health information and environmental insights.
