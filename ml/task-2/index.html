Introduction

Background

  • Quality inspection is a crucial step in tomato production and marketing
  • Physical attributes such as shape and color are key determinants of consumer acceptability
  • Current challenges in quality assessment:
    • Manual inspection is prone to human error
    • Inconsistencies in standardization
    • Incorrect classification and processing delays
    • Subjective evaluations due to fatigue
  • Improper sorting based on ripeness leads to:
    • Accelerated ripening due to ethylene release
    • Reduced shelf life
    • Increased susceptibility to diseases
    • Economic losses across the production chain

Research Objectives

This study aims to classify tomatoes based on their colorimetric characteristics, which influence consumer purchasing potential, using the decision tree algorithm. The goal is to provide an automated solution for tomato classification that can improve quality control and reduce losses in the agricultural industry.

"Automated selection methods offer a solution to limitations across agriculture, agroindustry, and retail sectors. Computer vision systems enhance inspection efficiency, reduce costs, and ensure more consistent standardization, improving the quality control process."

Advantages of Computer Vision

  • Fast and consistent evaluation of fruits
  • Elimination of human subjectivity
  • Ability to process large volumes of products
  • Integration with existing production lines
  • Quantitative measurement of quality parameters

Tomato Characterization

Tomato Classification System

Tomatoes were classified into two categories based on consumer purchasing potential:

  • Major Purchasing Potential (MPP): 20 fruits with mature red coloration (>90% red color)
  • Low Purchasing Potential (LPP): 40 fruits with either immature (green) or intermediate (mix of green and red) coloration
  • Classification aligned with CEAGESP standards (2003)
  • Fruits were sourced from the Italiano cultivar (Solanum lycopersicum L.) grown in greenhouses

Image Acquisition Setup

  • Digital camera (Canon Powershot G9 X RGB) mounted 0.24m above bench
  • Two 100W halogen lamps positioned at 0.84m height
  • White background to minimize shading effects
  • Images processed using ImageJ software to extract RGB values
  • Controlled environment to ensure consistent lighting and image quality

Color Model Conversion

RGB values were converted to additional color models for comprehensive analysis:

  • RGB Model: Red, Green, Blue intensity values
  • HSI Model: Hue (H) representing color (0¬∞ to 360¬∞)
  • CIELab Model:
    • Luminance (L): brightness (0 to 100)
    • Chromaticity a: green (-128) to red (+128)
    • Chromaticity b: blue (-128) to yellow (+128)

Consumer Preference Factors

For consumers, ripeness-related attributes such as color, texture, and firmness are critical indicators of quality. Specifically, red and uniform tomato coloration is a decisive factor for consumer acceptance. The study's classification into MPP and LPP categories was designed to align with these consumer preferences.

Data Acquisition & Preprocessing

Image Acquisition Pipeline

  1. Sample preparation: Tomatoes were cleaned and positioned individually on the measurement bench
  2. Image capture: Digital Canon Powershot G9 X RGB camera positioned 0.24m above bench
  3. Lighting control: Two 100W halogen lamps at 0.84m height provided consistent illumination
  4. Background preparation: White background to minimize shading effects
  5. Image digital storage: Images saved in high-resolution JPG format
  6. Quality control: Visual inspection of each image to ensure absence of reflections, shadows, or artifacts

Image Processing & Feature Extraction

  1. Software tool: ImageJ open-source software was used for all image processing
  2. Region of interest selection: Fruit area was isolated from the background
  3. RGB extraction: Average RGB values calculated from all pixels in the fruit region
  4. Color space conversion: RGB values mathematically transformed to:
    • HSI Model using rotation transformation (Pedrini & Schwartz, 2008)
    • CIELab Model using standardized color space equations
  5. Data organization: All extracted features compiled into a structured dataset
  6. Quality assurance: Statistical analysis conducted to check data integrity:
    • Outlier detection using z-scores
    • Normality validation with Shapiro-Wilk test
    • Measurement of central tendency and dispersion for all features

Decision Tree Model Implementation

Algorithm Implementation

  • Programming environment: R software (version 4.3.2)
  • Algorithm: Part (Recursive Partitioning and Regression Trees)
  • Implementation technique: CART (Classification and Regression Trees)
  • Split criterion: Gini index impurity measure

    Gini index = 1 - Œ£(pi)2

    Where pi is the probability of class i in a node

  • R packages used:
    • rpart - For decision tree implementation
    • caret - For model training and evaluation
    • e1071 - For statistical utilities
  • Pseudocode: 
    # Load libraries
library(rpart)
library(caret)

# Split data
set.seed(123)  # For reproducibility
trainIndex <- createDataPartition(tomato_data$class, p=0.75, list=FALSE)
training_data <- tomato_data[trainIndex,]
testing_data <- tomato_data[-trainIndex,]

# Train model with cross-validation for cp selection
control <- rpart.control(minsplit=2, minbucket=1, cp=0.001)
dt_model <- rpart(class ~ R + G + B + H + L + a + b, 
                  data=training_data,
                  method="class",
                  control=control)

# Prune tree
optimal_cp <- dt_model$cptable[which.min(dt_model$cptable[,"xerror"]),"CP"]
pruned_model <- prune(dt_model, cp=optimal_cp)

# Evaluate on test set
predictions <- predict(pruned_model, newdata=testing_data, type="class")
conf_matrix <- confusionMatrix(predictions, testing_data$class)

Hyperparameter Optimization

Complexity Parameter (cp) Optimization

The complexity parameter prevents overfitting by controlling tree growth. A smaller cp creates larger trees, while a larger cp creates smaller trees.

CP Value Tree Size Rel. Error Cross-Val. Error Standard Deviation
Inf 1 1.000 1.000 0.089
0.180 2 0.500 0.500 0.075
0.019 4 0.150 0.150 0.053

The optimal cp value of 0.019 was selected as it provided the lowest cross-validation error while maintaining a simple tree structure with just 4 nodes.

Other Training Parameters

  • minsplit: 2 (minimum number of observations in a node for it to be split)
  • minbucket: 1 (minimum number of observations in any terminal node)
  • maxdepth: 30 (maximum depth of any node in the tree)
  • usesurrogate: 2 (use surrogate splits to handle missing data)

Cross-Validation Strategy

  • 10-fold cross-validation used to evaluate model performance
  • Dataset partitioned into 10 equal subsets
  • Each fold used as validation set once while the remaining folds formed the training set
  • Performance metrics averaged across all 10 iterations
  • This approach provides a robust estimate of model generalization capability

Decision Tree Rule Set

Pseudocode Rule Representation

IF (G < 71) THEN
  IF (R < 86) THEN
    PREDICTION = "LPP"  # 7% of training samples
  ELSE
    IF (B < 19) THEN
      PREDICTION = "MPP"  # 2% of training samples
    ELSE
      PREDICTION = "LPP"  # 31% of training samples
    END IF
  END IF
ELSE
  PREDICTION = "LPP"  # 67% of training samples
END IF

Rule Interpretation

  • First decision point (root node) depends on green intensity (G)
  • Green intensity < 71 indicates potentially ripe fruits (needs further checking)
  • Green intensity ‚â• 71 directly classifies fruits as LPP (immature)
  • For potentially ripe fruits, red intensity (R) becomes the second decision factor
  • If red intensity < 86, fruits are classified as LPP (insufficient redness)
  • For fruits with adequate red intensity, blue intensity (B) provides final classification
  • Blue intensity < 19 indicates MPP (fully ripe tomatoes)

Biological Significance

  • Green intensity (G) represents chlorophyll concentration in fruit skin
  • Red intensity (R) corresponds to lycopene pigment accumulation during ripening
  • Blue intensity (B) relates to various minor pigments and tissue structure
  • The decision path mirrors the biological ripening process:
    • As tomatoes ripen, chlorophyll (green) degrades
    • Simultaneously, lycopene (red) content increases
    • Fine distinctions in ripeness relate to subtle color characteristics
  • Automated threshold detection approximates expert human judgment

Decision Tree Model

Decision Tree Structure

  • First node: G < 71 (Green intensity)
    • If G > 71, classify as LPP (33% of fruits)
    • If G < 71, continue to next node
  • Second node: R < 86 (Red intensity)
    • If R < 86, classify as LPP (7% of fruits)
    • If R > 86, continue to next node
  • Third node: B < 19 (Blue intensity)
    • If B < 19, classify as MPP (2% of fruits)
    • If B > 19, classify as LPP (27% of fruits)
  • 67% of fruits in training set were directly classified as MPP

Tree Pruning Process

  • Complexity parameter (cp) evaluates overfitting and limits tree growth
  • Optimal cp value: 0.019
  • This value corresponds to a tree with 4 nodes
  • Represents best balance between simplicity and accuracy
  • Prevents overfitting while maintaining classification power

Importance Analysis

  • Discrepancy between statistical significance and model importance: While statistical tests showed significant differences for almost all colorimetric characteristics, the decision tree algorithm selected only three (G, R, B) as the most effective predictors.
  • Primary discriminator: Green intensity (G) emerged as the most important feature, serving as the first decision node in the tree. This aligns with the largest mean difference between classes observed in the statistical analysis.
  • Secondary discriminators: Red (R) and Blue (B) intensities provided additional refinement in classification, particularly for fruits with lower green intensity values.
  • Non-selected features: Despite statistical significance, Hue (H), luminance (L), and chromaticities (a, b) were not included in the final model, suggesting redundancy or lower predictive power when used in combination with RGB components.
  • RGB vs. alternative color spaces: The decision tree's preference for RGB features over derived color spaces (HSI, CIELab) suggests that for this specific classification task, the direct color intensities provided sufficient discriminatory power without requiring conversion to perceptual color spaces.

Evolution of Tree-Based Methods

Early Tree Models

  • ID3 Algorithm (1986) - Iterative Dichotomiser 3, developed by Ross Quinlan
  • C4.5 (1993) - Extension of ID3 with improved handling of continuous attributes
  • CART (1984) - Classification and Regression Trees by Breiman et al.
  • Limited to simple decision boundaries
  • Prone to overfitting with complex data

Ensemble Methods

  • Random Forests (2001) - Multiple trees on bootstrapped samples
  • Gradient Boosting (1999) - Sequential improvement of weak learners
  • XGBoost (2016) - Optimized gradient boosting implementation
  • Higher accuracy than single trees
  • More robust against overfitting
  • Reduced interpretability compared to single trees

Recent Developments

  • Explainable AI (XAI) - Methods to interpret complex tree ensembles
  • Reinforcement Learning Trees - Adaptive tree growth
  • Fuzzy Decision Trees - Incorporating uncertainty in decisions
  • Neural-backed Decision Trees - Hybrid approaches
  • Focus on balancing performance with interpretability
  • Integration with deep learning approaches

Positioning of Current Study in ML Evolution

Our research leverages classic decision tree methodology (CART) with modern optimization techniques (cross-validation based pruning). The focus on interpretability and practical implementation aligns with contemporary trends in applied machine learning, where explainability is increasingly valued alongside raw performance metrics.

While ensemble methods might offer marginal improvements in classification accuracy, our study prioritizes model simplicity and clear decision rules that can be easily implemented in agricultural and food processing environments. This approach demonstrates how established machine learning techniques can be effectively applied to solve practical problems in agricultural engineering.
